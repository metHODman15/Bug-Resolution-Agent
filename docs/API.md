# API Documentation

This document describes the API contract between the VS Code extension and the Python backend service.

## Python Service Endpoints

The Python backend runs on `http://127.0.0.1:8000` and provides the following endpoints:

### POST /normalize
Normalizes business jargon to technical terminology.

**Request:**
```json
{
  "text": "string - business description with jargon"
}
```

**Response:**
```json
{
  "normalized": "string - technical description"
}
```

### POST /summarize
Performs local extractive summarization.

**Request:**
```json
{
  "items": ["string array - text items to summarize"],
  "summary_short": "boolean - true for short summary, false for detailed"
}
```

**Response:**
```json
{
  "summary": "string - summarized text"
}
```

### POST /search
Performs semantic vector search.

**Request:**
```json
{
  "query": "string - search query",
  "filters": "object - optional metadata filters"
}
```

**Response:**
```json
{
  "results": [
    {
      "text": "string - matched text",
      "score": "number - similarity score",
      "metadata": "object - additional data"
    }
  ]
}
```

### POST /build_prompt
Builds optimized prompts for LLM calls.

**Request:**
```json
{
  "type": "string - 'investigation', 'summary', or 'resolution'",
  "data": "object - prompt data"
}
```

**Response:**
```json
{
  "optimized_prompt": "string - ready-to-use prompt"
}
```

### POST /llm_call
Makes LLM calls with usage tracking.

**Request:**
```json
{
  "prompt": "string - optimized prompt"
}
```

**Response:**
```json
{
  "answer": "string - LLM response",
  "cited_ids": "array - referenced item IDs",
  "reasoning": "string - reasoning explanation",
  "usage": {
    "input_tokens": "number",
    "output_tokens": "number"
  }
}
```

### GET /tokens/session/{session_id}
Retrieves token usage for a session.

**Response:**
```json
{
  "total_tokens": "number - cumulative tokens"
}
```

## Extension ↔ Python Contract

### Data Flow
1. User input → Extension → Python `/normalize`
2. Normalized text → Python `/build_prompt`
3. Optimized prompt → Python `/llm_call`
4. Result + tokens → Extension → UI update

### Error Handling
- All endpoints return HTTP 500 on errors
- Extension displays user-friendly error messages
- Python service logs errors without exposing sensitive data

### Session Management
- Session IDs generated by extension
- Token tracking per session
- Automatic cleanup on extension restart

## Python NLP Pipeline

### Components
- **Normalization**: Fuzzy matching + taxonomy rules
- **Summarization**: TF-IDF based extractive methods
- **Embeddings**: SentenceTransformers + FAISS indexing
- **Prompt Building**: Template substitution + token limits
- **Token Tracking**: Session-based accumulation

### Configuration
- Taxonomy: `service_python/config/taxonomy.json`
- Templates: `service_python/config/prompt_templates.json`
- Environment: `.env` file for API keys
